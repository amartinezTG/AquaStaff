Descripción General del Sistema

AquaStaff es el sistema de gestión interno de Aqua Car Club, una cadena de autolavados de membresía ubicada en Chihuahua, México. Opera bajo un modelo de negocio híbrido: membresías recurrentes y servicios de pago por uso (paquetes).

El sistema es una aplicación web construida en Laravel 10 (PHP 8), desplegada en un servidor Windows Server con IIS. Se accede exclusivamente por personal interno (staff) autenticado mediante roles.

MóduloDescripción
DashboardMétricas de ventas en tiempo real, tráfico por hora, membresías activas
CajeroRegistro y consulta de transacciones, importación de archivos TPV Procepago
MembresíasGestión y análisis de membresías activas y renovaciones
IndicadoresReportes operativos y financieros con exportación a Excel/PDF
Corte de CajaArqueos de caja por sucursal con exportación PDF/Excel
Caja ChicaRegistro de gastos menores
Facturación GlobalGeneración de CFDI 4.0 agrupados por tipo de pago (FacturoPorTi)
Facturación IndividualGeneración de CFDI 4.0 individuales para transacciones sin folio físico
InventariosControl de productos y transferencias entre sucursales
PromocionesCreación y seguimiento de promociones vigentes
AdministraciónAuditoría de transacciones, detección de huecos, monitor de sincronización
Proyecto hermano: AquaFacturacion es el portal público para que los clientes generen su factura individual vía web. Comparte la misma base de datos MySQL y el mismo servidor IIS. Los archivos PDF/XML generados en AquaStaff se escriben también en el directorio de AquaFacturacion (dual-write).

Casos de Uso

CU-01 · Autenticación de Staff

El usuario ingresa su correo y contraseña. El sistema verifica credenciales contra la tabla staff_users y establece la sesión con su rol asignado.

RolIDAcceso
SUPERADMIN1Todo el sistema
ADMINISTRATOR2Todo excepto configuración avanzada
MANAGER3Cajero, Membresías, Indicadores, Corte de Caja, Inventarios

CU-02 · Consulta de Dashboard

Al ingresar, el sistema muestra métricas del día actual: total de transacciones, ventas, membresías vendidas, comparativa vs día anterior, desglose por hora, distribución por tipo de pago.

CU-03 · Consulta de Transacciones (Cajero)

El personal consulta el historial de transacciones filtrando por rango de fechas, tipo de transacción, forma de pago y cajero. Exportación disponible en Excel/CSV.

CU-04 · Importación de Archivo Procepago

Se carga el archivo Excel del TPV Procepago. El sistema parsea las filas e inserta los registros en procepago_pagos. Posteriormente se ejecuta análisis de conciliación para detectar discrepancias con local_transaction.

CU-05 · Generación de Factura Global

El contador selecciona un rango de fechas. El sistema agrupa transacciones no facturadas por tipo de pago y cajero, calcula base + IVA 8%, y genera un CFDI 4.0 vía API FacturoPorTi. Guarda UUID, serie, folio, PDF y XML.

CU-06 · Generación de Factura Individual

El staff selecciona transacciones individuales (renovaciones, compras de membresía) que no tienen CadenaFacturacion (folio físico). Ingresa o busca la cuenta fiscal del cliente, ajusta la fecha de emisión dentro de las 72 horas requeridas por el SAT, y genera el CFDI individual. Los archivos se guardan en ambos proyectos simultáneamente.

CU-07 · Corte de Caja

Al cierre del día, el cajero registra los totales de efectivo, tarjeta, cortesías y gastos por sucursal. El sistema genera el arqueo y permite exportarlo en PDF o Excel.

CU-08 · Gestión de Membresías

Consulta de membresías activas, ventas del período, distribución por tipo de paquete. Análisis de clientes recurrentes vs nuevos.

CU-09 · Indicadores Operativos

Generación de reportes mensuales con totales por cajero, por tipo de membresía, ingresos por proveedor (CARRERA / INTERLOGIC). Exportación a Excel y PDF.

CU-10 · Gestión de Inventario

Registro de productos, transferencias entre sucursales con seguimiento de estado (pendiente → entregado). Log de movimientos de inventario.

CU-11 · Auditoría de Transacciones

El sistema detecta huecos (gaps) en la secuencia de local_transaction._id por día, indicando posibles transacciones eliminadas o no sincronizadas. Exportación de reporte en CSV.

CU-12 · Monitor de Sincronización

Visualización del estado de sincronización del transactions_log entre proveedores y la base de datos central.

Diagramas de Flujo del Proceso

3.1 Generación de Factura Global

Contador selecciona rango de fechas
         ↓
Sistema consulta local_transaction
(sin global_invoice_id, PaymentType ≠ 3 Cortesía)
         ↓
Agrupa por PaymentType + Atm (cajero)
         ↓
Obtiene token FacturoPorTi (cache forever)
         ↓
Para cada grupo:
  Calcula  Base = Total / 1.08
  Calcula  IVA  = Total - Base
  Construye JSON CFDI 4.0
  POST /servicios/timbrar/json
         ↓
¿Éxito?
  SÍ → Guarda GlobalInvoice (UUID, serie, folio)
       Actualiza local_transaction.global_invoice_id
       Guarda PDF/XML en storage/app/public/invoices/
  NO → Retorna error al frontend (código + mensaje SAT)

3.2 Generación de Factura Individual

Staff filtra transacciones (por fecha, tipo, cajero, estatus)
         ↓
Selecciona una o varias transacciones
         ↓
Abre modal: ingresa RFC o busca cuenta fiscal
Sistema autocompleta datos desde fiscal_accounts (debounce 300ms)
         ↓
Staff verifica fecha de emisión
  Default = ahora - 3 minutos
  Validación JS: máximo 72 horas hacia atrás, no futuro (regla SAT)
         ↓
Confirma y envía al servidor
         ↓
Obtiene token FacturoPorTi (mismo cache compartido)
         ↓
Para cada transacción seleccionada:
  Nombre archivo = CadenaFacturacion ?: 'IND_' + local_transaction_id
  Construye CFDI individual (receptor = cuenta fiscal del cliente)
  POST /servicios/timbrar/json
         ↓
¿Éxito?
  SÍ → Actualiza local_transaction.fiscal_invoice = UUID
       Guarda PDF/XML → AquaStaff storage/app/public/pdfs|xmls/
       Guarda copia   → AquaFacturacion storage/app/public/pdfs|xmls/
  NO → Retorna error al frontend

3.3 Importación y Conciliación Procepago

Administrador carga archivo Excel (.xlsx) del TPV
         ↓
CajeroController::importarProcepago()
Lee filas con PhpSpreadsheet
         ↓
Valida columnas (folio, monto, fecha, referencia)
Inserta batch en procepago_pagos
         ↓
Análisis de conciliación (opcional):
  JOIN local_transaction vs procepago_pagos
  por num_operacion / _id
         ↓
Retorna discrepancias:
  ● Transacciones en local pero no en Procepago
  ● Montos diferentes entre ambos sistemas
  ● Transacciones en Procepago pero no en local

Regla SAT 72 Horas

Importante: El SAT requiere que la Fecha de emisión del CFDI no sea más de 72 horas anterior al momento del timbrado. Para transacciones antiguas (renovaciones de meses previos), el staff debe establecer la fecha de emisión al momento actual — nunca a la fecha de la transacción original.

Cálculo IVA 8% (Zona Fronteriza)

Base = Total / 1.08
IVA  = Total - Base

Ejemplo:
  Total = $216.00
  Base  = $200.00
  IVA   = $16.00
